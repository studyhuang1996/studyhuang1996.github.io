jsp内置对象

发表于 2017-07-13 | 分类于 jsp | 阅读次数

​ jsp中的内置对象

1 pageContext javax.servlet.jsp.PageContext 页面容器
2 request javax.servlet.http.HttpServletRequest 获取用户的请求
3 response javax.servlet.http.HttpServletResponse 服务器向客户端的回应信息
4 session javax.servlet.http.HttpSession 保存每个用户的信息
5 application javax.servlet.ServletContext 所有用户的共享信息
6 config javax.servlet.ServletConfig 服务器配置
7 out javax.servlet.jsp.JspWriter 页面输出
8 page java.lang.Object 从该页面表示出来的一个servlet实例
9 exception java.lang.Throwable 发生异常，错误页中起作用

​

4种属性范围

​ page:只在一个页面中保存属性，跳转之后无效。

​ <% pageContext.setAttribute("name","studyhuang");

pageContext.setAttribute("date",new Date()); %>

<% String username=(String)pageContext.getAttribute("name");

Date date=(Date)pageContext.getAttribute("date"); %>

<h2>用户名：<%=username%></h2>

<h2>日期：<%=date%></h2>

request:只在请求中保存，服务器跳转后依然无效。

<%@ page import="javax.servlet.http.HttpServletRequest"%>

` <% //设置属性在本页面，跳转后没办法获得

     request.setAttribute("name","studyhuang");
     request.setAttribute("req","在服务器的跳转中起作用");
%>`

session:在一次会话范围中，可以使用，新开浏览器失效。

不管客户端还是服务器只要设置了属性都可以获得。

application:在整个服务器上保存，所有用户可用。

hexo创建文章并部署

发表于 2017-07-11 | 阅读次数

背景：搭建博客已经有一段时间了，肚子里没墨，搭好就没怎么用了，暑假空闲时间一多，想着辛苦搭建的，得利用起来，发现忘了，so。。

1.创建文章

(1)hexo new ”文章标题“

(2)hexo new page “文件名”

2.找到SOURCE/_POSTS的文件夹下刚创建的文件，并用markdown编辑器打开文件，然后你就可以自由写 文章了

3.写好文章后，部署到github

(1) 先清除缓存

hexo clean

(2)部署

hexo g -d

4.刚开始还不咋会用markdown，见谅，写给健忘的自己。
