React入门学习总结（一）

发表于 2018-05-28 | 分类于 前端 | 阅读次数 
有了前面JavaScript和ES6的初步了解，真正进入了React的学习了。学习React之前，我们首先要搭建React的开发环境。开发环境的搭建中，我们需要先安装Node.js,可以到[官网](https://nodejs.org/en/)进行下载安装。安装完Node.js后，我们可以初始化一个React的项目。

1.安装React.js

由于React技术依赖太多的技术栈，为了避免搭个环境都搞半天，facebook提供一个快速开发React应用的工具，名为create-react-app。在控制台输入以下命令进行安装。

在浏览器自动打开地址为http://localhost:3000/的网页。到这，创建首个React应用完成。

2.React的设计思想

​ 通过组件来开发应用。组件是指能完成某个特定功能的独立的，可重用的代码。也就是说现在我们开发一个功能，就把这个功能都写在这个组件中，这个组件中包含html，css，js等，如果有有组件要用到他就直接引用。达到可重用性和独立的特点。组件只关注自己的功能。这和我们以前把html，css，js分开写不一样的地方。一个应用就由多个组件构造完成。

3.介绍下刚才自动生成的文件结构

src目录是源代码，我们只需关注这个。src/index.js是入口文件。

  1. JSX语法（JavaScript的语法扩展）

    jsx语法说白了就是在javaScript里我们可以直接嵌入html语言、，他会自动帮我们解析成js代码。但是其本质是javascript对象，在JavaScript每次遇到jsx时，会把他自动编译成javascript代码。。de

  2. 虚拟DOM(Virtual DOM)

    对dom进行模拟,会对比这一次产生的Virtual DOM和上次渲染的Virtual DOM的差别，修改有差别的地方。

  3. 组件(重点)

    • 组件名首字母必须大写 HelloMessage
    • 必须有render方法，输出组件
    • render 只能有一个顶层元素

  4. state和props

    • props是对外接口，state是内部状态
    • state

  5. 生命周期

    • 装载过程：组件第一次在DOM树渲染
    • 更新过程：组件被重新渲染
    • 卸载过程：组件从DOM中删除

  6. 组件的嵌套

    在实际的开发中，总不可能只有一个组件。一个应用总是由多个组件组合嵌套而成。

  7. proptypes

    1. 事件处理

      • React 事件使用驼峰命名，而不是全部小写。onClick
      • 通过 JSX , 你传递一个函数作为事件处理程序

    2. 表单
